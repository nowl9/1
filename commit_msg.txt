feat(recorder): widen --record-feeds with 3 capture-only latency streams

Adds three auxiliary, capture-only feed recorders that run ONLY under
--record-feeds, writing into the SAME gzipped-JSONL format the existing
recorder uses ({base}/{tag}/{date}/frames-HH.jsonl.gz) with the SAME outer
per-frame ts stamp the k-way replay merge keys on:

  * spot      -- Deribit deribit_price_index.btc_usd WS push (fast spot ref),
                 a dedicated connection so the deribit recording is unchanged.
  * chainlink -- Polygon BTC/USD aggregator latestRoundData via raw JSON-RPC
                 eth_call (no web3 dep); captures round_id/answer/updated_at
                 (the latency-critical on-chain push timestamp, not just price).
  * pm5min    -- Polymarket BTC 5-minute up/down binaries, discovered by the
                 btc-updown-5m slug prefix and polled at HIGH frequency (1s)
                 so the slow repricing lag is visible offline.

Capture-only guardrails: none of these flow into pricing, signals, gates, or
execution; the 5-minute contracts are recorded but NOT added to the arb's
tracked/signal universe (the live PolymarketFeed threshold filter still
rejects them -- no strike). Default live/paper/replay behavior is unchanged;
the new tasks start only when a recorder is present (--record-feeds, live
mode) and each self-disables on a blocked stream without touching the live
feeds. The replay reader ignores the new tags (its k-way merge keys on the
outer ts and falls to source-order 99 for unknown tags).

Recorder change: FrameRecorder.record now accepts a bare-string source tag in
addition to a DataSource enum; enum-source output is byte-identical to before.

Tests: +13 (decode incl. signed answer, discovery filter / Up-token
resolution, network-free discovery+book recording via httpx MockTransport,
each stream's on-disk format, replay-merge parses widened recordings,
DataSource byte-identity). Full suite 747 passed.
