fix(main): C4 clean shutdown -- one Ctrl-C banks every gz with a trailer; SIGBREAK wired

Completes the shutdown chain for the 2026-06-10 capture-trust work:
stop signal -> stop_event -> every task winds down (dashboard included
since C3) -> run() finally -> recorder.close() flushes and closes every
recording gzip with a valid trailer (close failures CRITICAL since C1).
This retires the final-hour truncation defect: long runs previously
ended in a force-kill because the run never exited, losing the last
hour's buffered gzip tail.

main.py:
* signal wiring factored into _install_stop_signals(stop_event); now
  also registers SIGBREAK (Windows Ctrl-Break) alongside SIGINT and
  SIGTERM.  Ctrl-Break follows the exact same clean-stop path as
  Ctrl-C, and is what a supervisor (or the Phase-3 drill) can deliver
  programmatically via GenerateConsoleCtrlEvent(CTRL_BREAK_EVENT).

Tests: 3 new for the helper (Windows signal.signal fallback path,
loop.add_signal_handler path, SIGBREAK -> stop_event).
Full suite: 868 passed, 0 failed.
