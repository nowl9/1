fix(recorder): C1 ENOSPC graceful-stop-loud + --record-feeds disk preflight

Root-cause fix for the 2026-06-10 silent capture stop (ENOSPC during a
polymarket write tripped the old disable-forever policy, silently
killing all six streams for 11 hours).

recorder.py -- failure policy revised from silent-disable to
graceful-stop-LOUD:
* _disable() replaced by _fail(): closes EVERY open gzip handle
  (banking buffered frames with valid trailers), logs CRITICAL
  frame_recorder.fatal, appends a RECORDER_FAILED sentinel
  (reason+ts+close results) under base_dir, and fires a new on_fatal
  callback exactly once so the orchestrator stops the run.
* close() / rotation-close failures now log CRITICAL with stream+path
  instead of bare `except: pass` (no more swallowed close errors).
* record() still never raises into feed callbacks.

main.py:
* _record_feeds_preflight(): --record-feeds refuses to START when the
  recording volume has < recorder_min_free_gb (default 20 GiB) free,
  measured via shutil.disk_usage on the nearest existing ancestor
  (never file-size sums).
* run() now returns an exit code (0 ok / 1 recorder fatal / 2
  preflight refused); on_fatal wires recorder failure -> stop_event +
  nonzero exit; main() sys.exit()s nonzero codes.

config.py: new recorder_min_free_gb knob (default 20.0).

Tests: failure-policy tests rewritten for the new policy (trailers
banked on all streams, sentinel content, on_fatal once, callback
exception contained, close-failure loudness); new preflight guard
tests. Full suite: 847 passed; the single red is the pre-existing
test_fresh_data_passes flake (signal path, out of scope; passes in
isolation).
