fix(main): C3 dashboard task honors stop_event -- --duration runs complete

Second defect from the 2026-06-10 incident review: _dashboard_task
awaited uvicorn server.serve() unconditionally (with uvicorn's own
signal handling disabled), so serve() never returned, the TaskGroup
never exited, --duration runs never completed, and NO run path ever
reached recorder.close() -- the mechanism behind the final-hour gzip
truncation on every long capture run and the --duration dead-end.

_dashboard_task now races serve() against stop_event.wait():
* stop_event fires -> server.should_exit = True, 10 s bounded grace,
  then hard cancel as a last resort; task returns and the TaskGroup
  can exit into run()'s finally, where recorder.close() banks every
  stream with valid gzip trailers.
* serve() crashing (e.g. port taken) is logged and returns, as before
  -- a dead dashboard neither wedges the group nor kills the run.

Tests: 3 new with a stubbed uvicorn.Server (no socket): stop honored +
should_exit set, crash path returns, duration->stop->dashboard-exit
chain completes. Full suite: 865 passed, 0 failed.
