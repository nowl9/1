feat(execution): un-short-circuit PM + book-walking fills (paper-trade step 2)

Build step 2 of docs/paper-trading-plan.md (sections 3.2, 4.1; Fork 1).
Routes Polymarket paper signals into the SAME FillSimulator Kalshi uses
and replaces top-of-book full-or-nothing with book-walking + partial
fills.  PM clears the identical 12-gate chain Kalshi does -- none skipped.

- orders.py: new venue-agnostic PaperExecutor (Fork 4 -- single executor,
  no per-venue protocol).  Submits nothing; flips PENDING -> PLACED; no-op
  refresh (the FillSimulator owns FILLED).  OrderManager.place() lifts the
  PM drop ONLY in paper mode (dry_run_paper_mode=True), routing PM to the
  PaperExecutor; in LIVE mode PM is still logged signal.polymarket_data_only
  and dropped -- the live-trading guardrail is untouched.

- fill_simulator.py: FillSimulator gains book_walk flag (default False =
  legacy top-of-book, preserved for existing callers/tests).  book_walk=True
  walks captured order_book_yes/no depth (ask-side, cheapest-first),
  accumulating at-or-below limit up to size_usd.  Outcomes are explicit and
  diagnosable: book_walk_full / book_walk_partial / empty_book /
  limit_below_book -- never a silent optimistic full fill on a thin/empty
  book.

- main.py: agent FillSimulator constructed with book_walk=True; new
  paper_orders_partial funnel counter; partial outcome counted in
  _record_paper_order.

Tests:
- test_fill_simulator.py (+10): full/VWAP/partial/empty/limit-below book
  walks, NO-side walk, defensive cheapest-first sort, ledger round-trip,
  and a regression that the DEFAULT simulator stays top-of-book.
- test_paper_pipeline.py: replaced the now-obsolete "PM short-circuit
  preserved" test with PM-routed-to-paper, PM-clears-same-gates-as-Kalshi
  parity, and PM-still-dropped-in-live-mode (guardrail).
