feat(dashboard): one-click Windows launcher (dashboard.bat)

P2 of the dashboard-launcher goal. The dashboard runs in-process with the
agent, so the whole startup is a single command. dashboard.bat wraps it for
double-click use from Explorer:

- cd to the repo root (%~dp0), default mode paper.
- Background-poll /api/health, then open the default browser at the SPA once
  it answers 200 (falls back to opening after ~30s).
- Foreground-run `py -3.12 -m btc_pm_arb.main`; closing the window (or Ctrl+C)
  tears the agent + dashboard down together. goto end guards stop the paper
  path from falling through into the replay block.
- `dashboard.bat replay` runs the headless deterministic replay (no dashboard,
  by design). "live" is intentionally NOT a launch option -- real-money is a
  guarded runtime toggle, not a CLI flag.
- HOST/PORT/token hoisted to the top so a later server deploy is a config
  change (plus the 127.0.0.1 bind in main.py), not a rewrite. Localhost only.

Tests: test_launcher.py -- static contract checks of the .bat (runs the right
module, polls health, opens the browser, binds localhost only, ASCII). Kept
network-free so they are deterministic under pytest-randomly. The runtime
"dashboard responds on localhost" invariant is covered by test_dashboard.py
(test_health / test_spa_served_at_root).

No trading behavior changed; the launcher only starts/stops the existing
entry point and reads state.
